# إعادة بناء النظام من الصفر

هذه التعليمات مخصصة لإعادة إنشاء قاعدة البيانات بالكامل من دون بيانات قديمة، بهدف التأكد من أن النظام قادر على إنشاء شركة جديدة بدون مشاكل.

## الملفات التي يجب تشغيلها بالترتيب

1. `services/master_setup.sql`
   - يحذف المخطط الحالي ويعيد إنشاء الجداول والدوال الأساسية
   - هذا الملف يمحو أي بيانات حالية

2. `services/deploy_all_functionss.sql`
   - يُنشئ أو يُحدِّث كل الدوال (RPCs) الخاصة بالنظام
   - يتضمن دالة `get_over_limit_customers` والدوال المالية

3. `services/setup_rls.sql`
   - يفعِّل سياسات RLS ويضبط الحماية لكل جدول
   - يضمن أن الشركة والمستخدمين يعملون بعزل صحيح حسب `organization_id`

## خطوات التنفيذ في Supabase SQL Editor

1. افتح مشروعك في Supabase Dashboard
2. انتقل إلى SQL Editor
3. افتح ملف `services/master_setup.sql`
4. انسخ كل المحتوى منه
5. الصق المحتوى في SQL Editor واضغط Run
6. بعد الانتهاء بنجاح، نفِّذ نفس الخطوات مع `services/deploy_all_functionss.sql`
7. ثم نفِّذ `services/setup_rls.sql`

## ملاحظات مهمة

- إذا كنت تريد اختبار النظام بدون بيانات سابقة، فهذا هو الإجراء الصحيح.
- بعد التشغيل يجب أن يكون البرنامج جاهزاً
- إذا ظهرت أي أخطاء، انسخ رسالة الخطأ بالكامل وأرسلها لي

## التحقق بعد التنفيذ

1. أنشئ شركة جديدة من واجهة البرنامج
2. أنشئ مستخدماً جديداً للعميل
3. تحقق من إنشاء دليل الحسابات
4. جرِّب إنشاء قيد محاسبي أو فاتورة

إذا أردت، أستطيع أيضاً كتابة ملف `rebuild_full_system.sql` واحد يدمج المحتوى الكامل للثلاثة ملفات في ترتيب صحيح.
