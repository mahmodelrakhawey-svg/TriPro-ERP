# إعادة بناء النظام من الصفر

هذه التعليمات مخصصة لإعادة إنشاء قاعدة البيانات بالكامل من دون بيانات قديمة، بهدف التأكد من أن النظام قادر على إنشاء شركة جديدة بدون مشاكل.

## الملفات التي يجب تشغيلها بالترتيب

1. `services/master_setup.sql`
   - يحذف المخطط الحالي ويعيد إنشاء الجداول والدوال الأساسية
   - تم تحديثه ليشمل جداول التصنيع الأساسية لضمان التجانس.

2. `services/update_system_stabilization.sql`
   - يضبط القيود، التريجرات، ومنطق الحسابات الآلي (مثل ترحيل المخزون).
   - يجب تشغيله قبل الدوال لضمان وجود الهيكل البرمجي.

3. `services/deploy_all_functionss.sql`
   - يُنشئ أو يُحدِّث كل الدوال (RPCs) الخاصة بالنظام
   - يدمج دوال النظام العام مع دوال التصنيع لضمان نقطة انطلاق واحدة.

4. `services/setup_rls.sql`
   - يفعِّل سياسات RLS ويضبط الحماية لكل جدول
   - يدعم "المستخدم العالمي" (Super Admin) للوصول لكافة المنظمات.

### 🏭 مديول التصنيع (MFG)
توجد ملفات إضافية في `services/mfg/` تتبع نفس الترتيب عند الحاجة لتحديث التصنيع منفصلاً.

## خطوات التنفيذ في Supabase SQL Editor

1. افتح مشروعك في Supabase Dashboard
2. انتقل إلى SQL Editor
3. افتح ملف `services/master_setup.sql`
4. انسخ كل المحتوى منه
5. الصق المحتوى في SQL Editor واضغط Run
6. بعد الانتهاء بنجاح، نفِّذ نفس الخطوات مع `services/deploy_all_functionss.sql`
7. ثم نفِّذ `services/setup_rls.sql`

## ملاحظات مهمة

- إذا كنت تريد اختبار النظام بدون بيانات سابقة، فهذا هو الإجراء الصحيح.
- بعد التشغيل يجب أن يكون البرنامج جاهزاً
- إذا ظهرت أي أخطاء، انسخ رسالة الخطأ بالكامل وأرسلها لي

## التحقق بعد التنفيذ

1. أنشئ شركة جديدة من واجهة البرنامج
2. أنشئ مستخدماً جديداً للعميل
3. تحقق من إنشاء دليل الحسابات
4. جرِّب إنشاء قيد محاسبي أو فاتورة

## 🏠 التشغيل المحلي (للتطوير بدون انترنت)

لتجربة التعديلات بعيداً عن بيانات العميل وبدون قيود الباقة المجانية:

1. تأكد من تشغيل **Docker Desktop**.
2. افتح المجلد في Terminal ونفذ:
   ```bash
   npx supabase start
   ```
3. استخدم الروابط الناتجة لتحديث ملف `.env.local`.
4. افتح `http://localhost:54323` (النسخة المحلية من Supabase).
5. نفذ ملفات الـ SQL المذكورة أعلاه في البيئة المحلية أولاً.

**نصيحة:** عند الانتهاء من العمل المحلي، استخدم `npx supabase stop` لتوفير موارد الجهاز.

## 💾 أخذ نسخة احتياطية من بيانات العميل (Production Backup)

**ملاحظة هامة:** يتطلب تنفيذ أوامر النسخ الاحتياطي تشغيل برنامج **Docker Desktop** على جهازك أولاً.

خطوات الحفاظ على بيانات العميل:

1. افتح Terminal ونفذ أمر الربط (مرة واحدة):
   ```bash
   npx supabase link --project-ref pjvphxfschfllpawfewn
   ```
2. نفذ أمر سحب البيانات الفعلي:
   ```bash
   npx supabase db dump --data-only -f client_backup.sql
   ```

**تحذير:** احتفظ بملف `client_backup.sql` بعيداً عن مجلدات النشر العامة.

## 💡 طرق بديلة للنسخ الاحتياطي (بدون Docker)

1. **تصدير سريع (CSV):** من واجهة Supabase -> Table Editor -> اختر الجدول -> Export CSV.
2. **أدوات الواجهة (GUI):** استخدم برنامج **DBeaver** واربطه بقاعدة البيانات باستخدام Connection String من الإعدادات، ثم استخدم خاصية Backup.
3. **أمر مباشر (pg_dump):** يتطلب تثبيت أدوات PostgreSQL أولاً. إذا كنت تستخدم Windows، يمكنك تثبيتها بفتح Terminal (كمسؤول) وتشغيل:
   ```powershell
   winget install PostgreSQL.PostgreSQL
   ```
   *ملاحظة: يجب إغلاق الـ Terminal وفتحه بعد التثبيت.*



إذا أردت، أستطيع أيضاً كتابة ملف `rebuild_full_system.sql` واحد يدمج المحتوى الكامل للثلاثة ملفات في ترتيب صحيح.
